# Agents

## Output Specifications

All outputs go to `workspace/`. The following deliverables are expected:

| File | Stage | Description |
|------|-------|-------------|
| `rebuttal_strategy.md` | Stage 0 | Prioritized concern list, experiment schedule, response strategy |
| `rebuttal_draft_v1.md` | Stage 1 | First complete draft addressing all reviewer concerns |
| `rebuttal_final.md` | Stage 2 | Final polished rebuttal ready for submission |
| `revision_plan.md` | Stage 2 | Camera-ready revision checklist for post-acceptance |

## Word Limit Tracking

The rebuttal has a strict **4000-word limit**. Track word count at every draft stage. If the draft exceeds the limit, prioritize compression of lower-priority responses (e.g., R1 acknowledgments, minor issues) while preserving full detail on high-priority items (R2/R4 concerns).

## Communication Tone Guidelines

### Email Messages
- Concise, action-oriented
- Use bullet points for status updates
- Include context for recipients who may not be following day-to-day progress
- Use clear subject lines

### Rebuttal Text
- Begin each reviewer section with thanks
- Structure: concern summary, response, evidence/data, commitment to revision
- Use "We" consistently (not "I")
- Cite specific numbers and table references

## Experiment Summary Output

When reporting supplementary experiments, include structured data:
- Experiment name, configuration, dataset, metric, result
- Comparison with paper-reported values
- Reviewer concern addressed
